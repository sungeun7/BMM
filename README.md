# 부루마블 (모바일)

핸드폰에서 즐기는 2~4인용 부루마블 스타일 보드게임입니다. 친구와 대전하거나 컴퓨터와 2~4인(팀전 포함) 플레이가 가능합니다.

## 실행 방법

1. `index.html` 파일을 브라우저에서 열기 (더블클릭 또는 브라우저로 드래그)
2. **로컬 서버** (같은 Wi‑Fi에서만 접속 가능):
   ```bash
   npm start
   ```
   휴대폰에서 `http://[PC IP]:3000` 접속

3. **같은 와이파이가 아니어도 실행 (어디서나 접속)**  
   서로 다른 Wi‑Fi·다른 지역에서도 플레이하려면, 게임을 **인터넷에 올리면** 됩니다. 아래 방법 중 하나로 하면 **HTTPS 주소**가 생기고, 그 주소를 친구에게 보내면 같은 와이파이가 아니어도 접속해 **온라인 대전**할 수 있습니다.

   **방법 A – Netlify Drop (가장 간단)**  
   1. [app.netlify.com/drop](https://app.netlify.com/drop) 접속  
   2. 이 프로젝트 **폴더 전체**를 드래그해서 올리기 (또는 압축(zip) 후 올리기)  
   3. 나오는 주소(예: `https://xxxx.netlify.app`)를 친구에게 전달  
   4. 둘 다 그 주소로 접속 → **친구와 대전** → **온라인 (다른 와이파이)** → 방 만들기 / 코드로 참가

   **방법 B – GitHub Pages**  
   1. 이 폴더를 GitHub 저장소에 올린 뒤  
   2. 저장소 **Settings → Pages**에서 Source를 해당 브랜치로 설정  
   3. `https://[사용자명].github.io/[저장소명]/` 주소로 접속 후, 위 주소를 친구에게 공유

4. **APK로 설치 (Android)**  
   휴대폰에 앱처럼 설치하려면 **Capacitor**로 APK를 만듭니다.  
   자세한 절차는 **[BUILD-APK.md](BUILD-APK.md)** 를 참고하세요. (Node.js + Android Studio 필요)

5. **앱처럼 사용 (APK 없이 홈 화면 설치)**  
   이 프로젝트는 PWA로 설치할 수 있습니다.  
   - Android Chrome: 메뉴(⋮) → **홈 화면에 추가** 또는 **앱 설치**  
   - iPhone Safari: 공유 버튼 → **홈 화면에 추가**  
   설치 후 아이콘으로 실행하면 브라우저 탭이 아닌 앱처럼(standalone) 실행됩니다.

### 접속이 안 될 때

**휴대폰에서 게임 페이지가 안 뜨는 경우**

1. **PC에서 서버 실행**  
   프로젝트 폴더에서 터미널(명령 프롬프트)을 열고 `npm start` 또는 `npx serve . -p 3000` 실행.

2. **PC IP 주소 확인**  
   - Windows: `ipconfig` 입력 후 **IPv4 주소** 확인 (예: 192.168.0.5)  
   - 휴대폰 브라우저에 `http://192.168.0.5:3000` 입력 (본인 PC IP로 바꾸기).

3. **같은 Wi‑Fi**  
   PC와 휴대폰이 **같은 Wi‑Fi**에 연결되어 있어야 합니다.

4. **방화벽**  
   접속이 안 되면 Windows 방화벽에서 **포트 3000** 허용 또는 Node 관련 앱 허용 후 다시 시도.

**온라인 대전(방 만들기·참가)이 안 되는 경우**

- 온라인 대전은 **HTTPS** 주소에서만 동작합니다. **같은 와이파이가 아니어도** 하려면 위 "방법 A – Netlify Drop"으로 게임을 올린 뒤, 나온 **HTTPS 주소**로 둘 다 접속하면 됩니다.
- 방 코드가 맞는지, 호스트가 "방 만들기" 후 대기 중인지 확인하세요.
- 그대로 안 되면 "같은 기기" 또는 "vs 컴퓨터" 모드로 플레이할 수 있습니다.

## 플레이 모드

- **친구와 대전**
  - **같은 기기**: 2인 / 3인 / 4인 선택. 4인일 때 **2대2 팀전** 옵션 가능. 한 기기에서 번갈아 플레이.
  - **온라인 (다른 와이파이)**: 서로 다른 네트워크에서도 2인 대전 가능. **방 만들기**로 6자리 방 코드를 만들고, 상대는 **코드로 참가**에 코드를 입력해 접속. 호스트가 **게임 시작**을 누르면 시작. (PeerJS 기반 P2P 연결)
- **vs 컴퓨터**: 2인 / 3인 / 4인 선택 가능. 4인일 때 팀전 선택 가능. **플레이어 1만 사람**, 나머지는 컴퓨터.

## 보드 구성 (40칸)

- **출발**: 오른쪽 아래에서 시작, 시계 반대 방향으로 진행.
- **첫 번째 줄 (아래쪽 변)**: 충청·강원 (원주, 충주, 강릉, 천안, 청주, 세종 등)
- **두 번째 줄 (왼쪽 변)**: 전라·제주도 (순천, 여수, 광주, 전주, 목포, 군산 등)
- **세 번째 줄 (위쪽 변)**: 경상도 (진주, 창원, 부산, 대구, 울산, 경주 등)
- **네 번째 줄 (오른쪽 변)**: 서울·경기 (김포, 의정부, 고양, 성남, 수원, 인천, 서울 등)

각 줄의 **3번째 칸**은 황금카드, **6번째 칸**은 세금입니다.
이동 시 **한 칸씩 점프하는 애니메이션**으로 보드 위를 따라갑니다. 보드 중앙에는 한국 지도(`우리나라.jpg`)가 표시됩니다.

## 게임 방법

- **주사위 굴리기** 버튼 또는 **스페이스바**로 주사위를 굴립니다. (컴퓨터 턴에는 버튼 비활성화)
- **출발**을 지나치거나 도착할 때마다 월급 **₩20만**을 받습니다.
- **땅**에 도착하면 구매하거나 패스할 수 있습니다. 다른 플레이어(팀전이면 상대 팀) 땅이면 통행료를 냅니다.
- **한 줄을 모두 소유**하면 해당 줄 땅 통행료가 **30%** 올라갑니다.
- **황금카드**에 도착하면 운에 따라 돈을 받거나 내게 됩니다.
- **세금** 칸에서는 지정된 금액을 납부합니다.
- **무인도**에 도착하면 **3턴** 쉬어갑니다.
- **더블**(두 주사위가 같은 수)이 나오면 **한 번 더** 굴립니다.
- **금액이 부족**할 때(통행료·세금·땅 구매) **땅 판매**로 마련할 수 있습니다. 땅·건물 판매 시 **가격의 70%**를 받습니다.
- **자기 땅**에 도착하면 **건물**을 지을 수 있습니다.

### 건물

| 건물 | 건설 비용 (땅값 기준) | 통행료 보너스 |
|------|------------------------|----------------|
| 별장 △ | 50% | +50% |
| 빌딩 □ | 70% | +70% |
| 호텔 ★ | 90% | +90% |

한 땅당 별장 → 빌딩 → 호텔 순으로 하나씩만 지을 수 있습니다.

## 팀전 규칙

- 4인일 때 **2대2 팀전**을 선택하면 **팀 A(플레이어 1, 2)** vs **팀 B(플레이어 3, 4)**로 진행됩니다.
- 턴 순서는 **팀 A → 팀 B**가 번갈아가며 한 턴씩 진행됩니다. (1 → 3 → 2 → 4 → 1 → …)
- 한 팀이 먼저 전원 파산하면 상대 팀이 승리합니다.

## 승리·파산

- 돈이 0원 이하가 되면 파산합니다.
- **개인전**: 마지막까지 남은 한 명이 승리.
- **팀전**: 상대 팀이 먼저 전원 파산하면 승리.
- vs 컴퓨터에서 플레이어 1이 마지막에 남으면 **승리**, 컴퓨터가 남으면 **컴퓨터 승리**로 표시됩니다.

## 기타

- **효과음**: 주사위, 구매, 통행료, 월급, 황금카드, 무인도, 승리 시 Web Audio API로 재생됩니다.
- **메뉴로 돌아가기**: 게임 화면 상단 버튼으로 시작 화면으로 돌아갈 수 있습니다.

## 구성 파일

- `index.html` – 게임 화면 (시작 화면, 보드, 주사위, 플레이어 패널)
- `style.css` – 모바일 터치에 맞춘 스타일 (11×11 그리드 보드)
- `game.js` – 주사위, 이동, 땅/건물/통행료/황금카드/팀전/컴퓨터 AI 등 게임 로직
- `우리나라.jpg` – 보드 중앙에 표시되는 한국 지도 이미지

모바일 브라우저(Chrome, Safari 등)에서 열어 두고 번갈아가며 플레이하면 됩니다.
